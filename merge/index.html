How merging works here

When you select your PDF files, they're loaded directly into your browser's memory using the File API — they are never sent over the network. A PDF-processing library running locally in the page reads each file's internal page structure and copies the pages into a new combined document, all within your device's own processing power. Once the merge finishes, the result is offered to you as a direct download; no copy of any file passes through an external server at any point.

Frequently asked questions

Is it safe to merge confidential PDFs with this tool?

Yes. The merge happens entirely inside your browser using your device's own processing power. Your files are never uploaded to any server, so there is no remote copy created at any point.

Is there a limit to how many PDFs I can merge?

There is no artificial limit imposed by this tool. The practical limit is your device's available memory, since all processing happens locally rather than on a server.

Can I change the order of the pages before merging?

Yes. Reorder the files in the list above before merging — the final document follows the order shown, top to bottom.
