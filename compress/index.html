How compression works here

Your PDF is loaded directly in your browser using the File API — it's never uploaded anywhere. Each page is rendered onto a canvas at your device's own processing power, then re-encoded as a compressed JPEG image at the quality level you choose, and reassembled into a new PDF. This approach trades the original vector text and graphics for a smaller, image-based version of each page, which is why it's most effective on documents that were scans or photos to begin with.

Frequently asked questions

Why didn't my PDF get much smaller?

Compression works by re-rendering each page as an optimized image, so it helps most with PDFs that already contain large images or scanned pages. A PDF that is mostly plain text is usually already small and may not shrink much further.

Will compressing make my text unselectable?

Yes. Because this method re-renders each page as an image, the output PDF will not have selectable or searchable text, even if the original did. This is a tradeoff worth knowing about before compressing text-heavy documents.

Which quality setting should I use?

Balanced works well for most documents you plan to email or upload. Use High Quality if the document includes detailed images you want to preserve, and Maximum Compression only when file size matters more than visual quality, such as for quick previews.
